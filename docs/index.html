Installation

Prerequisites

  1. Visual Studio Code or Webstorm

  2. Have Node.js Installed

  3. Install Git (Group Use)

  4. Clone repository

  5. Navigate to directory and install expo globally

npm install -g expo-cli

  1. Start Python Server

  2. Start Application

    npm start

Next step is for emulator. Expo also let's you run it off lan if you install the android/ios application
NOTE: ios emulator devices will only work on ios devices with xcode

  1. Sync capacitors to devices

    npx cap sync ios

    or

    npx cap sync android

  2. Open project on xcode or Android Studio

    npx open ios

    or 

    npx open Android
  • If you get an error, run this command instead:
    npx open <device name>

Release Notes

Description

  • Xplore (working title) aims to become a social media platform centered around community engagement and social interaction. At its core lies the real-world, interactive treasure hunting application that you will develop over the course of this semester. The resulting treasure hunting experience will provide users with a new means for interacting with local businesses and community members.

Working Submission

  • Currently our project is finished with milestone 1 displaying "Hello World" on the main screen. We have already moved into our milestone 2 by adding the map features to our appliction with the start of our player location.

Milestone 2

  • For this milestone we decied to change how we were going to be building the application. Instead of using Ionic and Leaflet for the map we are changing to react native with google maps. We have a working geolocation tracking system with the google maps. As well we have stock holder location for treasures. We also created another repository that will handle out backend that uses Phyton Django. Started to work on the database and will have that conntected to the front end by milestone 3

Milestone 3

  • For milestone 3, we setup the skeleton of geofencing along with a UI rework for the hunt list. The geofencing works by drawing a circle range around the hunt that indicates the border of the hunt. When the user enters the diameter, it will trigger the hunt's start showing each of the clues (not implemented yet). The UI rework consisted of removing the list/map toggle button and switching to a scroll-view overlay that focuses on the hunts as the user scrolls through. There are still some QoL changes to these features that will be added.

Milestone 4

  • For this milestone we implemented a working login screen. Beforehand it was just a skeleton that would accept any input. Now users can sign in or register an account with the help of our database. Inputing a wrong password or failure to register shows an error (register error still being implemented).
